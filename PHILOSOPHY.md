# Philosophy

This document defines the permanent principles that guide all work in this workshop.

## Ownership over access

Users buy tools once and keep them. No cloud dependency. No remote activation or verification. No expiration dates.

Software should behave like a physical tool: purchased, owned, and usable regardless of the seller's business decisions.

## Offline first

All software built here must function fully without internet access. Internet access may be optional for specific features (such as importing reference images), but core functionality must never require it.

If a feature cannot work offline, it does not belong in the tool.

## Stability over novelty

Breaking changes are avoided. Updates are optional. Long-term reliability matters more than rapid iteration.

We prioritize boring, proven solutions over trendy frameworks. Dependencies should be minimal, stable, and justified.

## Restraint and scope discipline

Fewer features, deeply considered. No speculative additions. No "platform thinking."

Each project has a defined v1 scope. Features outside that scope are explicitly excluded, not deferred.

Feature creep is treated as a design failure.

## Software as a tool, not a service

The tools built here are finished objects, not evolving platforms. They are products, not services.

A tool should:

- Solve a specific problem completely
- Require no ongoing relationship with its creator
- Age gracefully without maintenance
- Store data in open, readable formats
- Respect the user's attention and focus

This is not a model optimized for recurring revenue. It is a model optimized for trust.

## What we reject

- Subscription models
- Account systems
- Analytics and telemetry
- Cloud storage requirements
- Automatic updates that cannot be disabled
- Proprietary file formats
- Dark patterns
- Growth-focused design

These are not temporary constraints. They are permanent boundaries.

## Why this matters

Creative work requires focus, trust, and stability. Tools that disappear, break, or change without warning harm the creative process.

This workshop exists to build tools that respect the people who use them.
